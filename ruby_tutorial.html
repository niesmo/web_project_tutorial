Introduction

This is a tutorial intended for developers who have a basic knowledge of programing including loops, variables, classes, and datatypes. Prior knowledge of Ruby programming is not necessary for these tutorials.

This set of tutorials assume that you already have Ruby installed on your machine and is ready to use. If you do not have ruby installed on your computer, please follow this install link before getting started with these tutorial

Ruby is an Object Oriented Programming Language. In Ruby everything is an object. Characters, numbers, arrays, and everything else is just an object in Ruby.

Ruby Interpreter (IRB)

IRB stands for Interactive Ruby which executes Ruby Code interactively.

Calling Some Functions

We talked about how everything is an object in Ruby. Therefore, in order to work with objects, we must call their public methods and that is how we interact with them. Here are some examples.

str = "this is an awesome Tutorial"
puts str.capitalize
puts str.length
puts str.slice 11, 7
puts str.slice(11,7).capitalize
puts str[11,7]

puts str.insert(18, "-X")
puts str.reverse

my_num = 12345

But again, everything is an object, there is not really a primitive type in Ruby! It is all objects and classes. That means that even numeric values are object. If that is the case, then the operations that we do on numberic values are functions. Take a look at these examples

my_num = 5
puts my_num.*(3) # my_num*3
puts my_num.+(5) # my_num+5
puts my_num.-(4) # my_num-4
puts my_num./(2.5) # my_num/2.5

These are called syntactic sugar that Ruby has added for the ease of use.

Methods

Methods are probably the most used feature of any programming language ever existed. You can create methods using the following syntax

def method_name [( [arg [= default]]...[, * arg [, &expr ]])]
    expr..
end

Your methods name must start with a lowercase letter. It is best practice that your method names are all in lowercase and separated with underscore if multiple words are used.

Note that the last expression in a method is returned unless an explicit return is specified

def hi
    "Hi"
end
    
puts hi
Try it

We can have methods that take in arguments as well

#method with 1 argument
def hi name
    "Hi " + name
end

puts hi "Mary"


#method with 2 arguments
def hi name,msg
    "Hi " + name + "\n" + msg
end

puts hi "David", "Welcome to the best tutorial site ever made"
Try it

This may be a strange concept, but we can have methods that take in as many arguments as we pass it to. These types of methods do exist in other programming languages such as C/C++. printf and scanf are two of the most popular examples of such functions. In Ruby we use an operator called the Splat Operator to indicate such thing. Here is an example

def avg(*num)
   sum = 0
   count = 0
   if num.respond_to?("each")
        num.each do |i|
           sum += i 
           count += 1
        end
    else
        count = 1
        sum = num
   end
   sum/count
end
puts avg 1,2,3
puts avg 2,3,4,5,6
puts avg 1

Classes

Classes are objects consisting of methods and variables. Classes can be instantiated to create new objects of that class. 

#A simple class
class dog #class name is dog

    @age, @color, @name
    
    def bark
        puts("wooof")
    end
end

frodo = Dog.new #creating a Dog object
frodo.bark()

In this example you can see the use of multiple instance variables, those that are unique to a specific object that is created from a class. There's also a simple instance method, which can only be called once a Dog object has been created. However you can also create an object passing values, use static methods, and there's 4 other types of variables in a class. Class, Global, Local, Constants. 

$global = 5 # global 

class Dog 
    LEGS = 4 #Constant
    @age, @color, @name
    @@numDogs = 0 #class variable

    def initialize(a, c, n) #constructor
        @age, @color, @name = a, c, n
        @@numDogs += 1
    end

    def self. legs
        totalLegs = LEGS * numDogs #totalLegs is local
        return numDogs
    end

    attr_accessor : age #accessor for creating get/set 
end

thorin = Dog.new(89, Grey, "Oaken'sheild")

You can also use basic Ruby syntax to create get and set methods. Methods that will allow you to access and change variables in a class. These are automatic using attr_accessor and it's sister methods.

Modules

Modules are a way of grouping different things all together. You can group classes, methods, and constants. One of the biggest advanteges is that they prevent from name clashing. You can have a class with the same name in two different modules without any problems

module Identifier
    statement1
    statement2
    ..................
end

For example, you can have a module that is a wrapper for all your simple mathematic functions

module MyMathModule
    PI = 3.1415
    def MyMathModule.square a
       a*a
    end
    def MyMathModule.pythag(a,b)
        ((a*a)+(b*b))**0.5
    end
end

Arrays

arr2 = Array.new 7
arr3 = arr1[2..5]
arr4 = arr3 + arr1

    arr.push(6)         #=> [0,1,2,3,4,5,6]
