Variables

Variables are simply memory locations that hold data in our program. Variables in Ruby are similar to variables in other programming languages.

In this section, we are going to talk about Global variables and Constants. If you would like to know more about other kinds of variables, you could go and read the Classes section of this tutorials.


Global Variables

Global variables are a kind of variables that you can access them from anywhere in your program. They must start with $. It is best practice that your global variables are all in lowercase and separated with underscore if multiple words.

$total_function_call_count = 0

def add a,b
    $total_function_count+=1
    a+b
end

def sub a,b
    $total_function_count+=1
    a-b
end

def mult a,b
    $total_function_count+=1
    a*b
end

def div a,b
    $total_function_count+=1
    a/b
end

add 1,2
sub 2,1
mult 8,9
div 2,1

puts "How many funciotn calls did we have ? #{$total_function_count}"

Constants

Constants are variables that are intended to stay the same all throughout the execution of a program. One great example for constant is the PI number. The number PI should NOT change under any circumstances.

Constant variables must begin with uppercase letters. It is best practice that the constant variable is in all uppercase. Re-assignign value to a constant will result in a warning (It still does the re-asignment however). It is worth noting that accessing a constant that has not been initialized yet will result in a runtime error. Another thing that is noticable is that you cannot assign a value to a constant variable from within a method. That is considered to be Dynamic Constant Assignment which will result in an error.

#constant variable
PROGRAM_NAME = "Best Tutorials"

def print_program_name
    puts PROGRAM_NAME
end

#setting a constant in a variable => ERROR
=begin
def set_program_name name
    PROGRAM_NAME = name
end
=end

print_program_name #=> will execute fine

Methods

Methods are probably the most used feature of any programming language ever existed. You can create methods using the following syntax

def method_name [( [arg [= default]]...[, * arg [, &expr ]])]
    expr..
end

Your methods name must start with a lowercase letter. It is best practice that your method names are all in lowercase and separated with underscore if multiple words are used.

Note that the last expression in a method is returned unless an explicit return is specified

def hi
    "Hi"
end
    
puts hi

We can have methods that take in arguments as well

#method with 1 argument

    "Hi " + name

puts hi "Mary"


#method with 2 arguments
def hi name,msg
    "Hi " + name + "\n" + msg
end

puts hi "David", "Welcome to the best tutorial site ever made"

This may be a strange concept, but we can have methods that take in as many arguments as we pass it to. These types of methods do exist in other programming languages such as C/C++. printf and scanf are two of the most popular examples of such functions. In Ruby we use an operator called the Splat Operator to indicate such thing. Here is an example

def avg(*num)
   sum = 0
   count = 0
   if num.respond_to?("each")
        num.each do |i|
           sum += i 
           count += 1
        end
    else
        count = 1
        sum = num
   end
   sum/count
end
puts avg 1,2,3
puts avg 2,3,4,5,6
puts avg 1

$global = 5 # global 

class Dog

    def self. legs
        totalLegs = LEGS * numDogs #totalLegs is local

Modules

Modules are a way of grouping different things all together. You can group classes, methods, and constants. One of the biggest advanteges is that they prevent from name clashing. You can have a class with the same name in two different modules without any problems

module Identifier
    statement1
    statement2
    ..................
end

For example, you can have a module that is a wrapper for all your simple mathematic functions

module MyMathModule
    PI = 3.1415
    def MyMathModule.square a
       a*a
    end
    def MyMathModule.pythag(a,b)
        ((a*a)+(b*b))**0.5
    end
end

Hashes

A Hash is a collection of key-value pairs like this: "employee" => "salary". It is similar to an Array, except that indexing is done via arbitrary keys of any object type, not an integer index.

The order in which you traverse a hash by either key or value may seem arbitrary and will generally not be in the insertion order. If you attempt to access a hash with a key that does not exist, the method will return nil.

You can create hashes in many different ways. Two of the most commons ways are shown below

my_hash = Hash.new
my_hash = Hash.new 0 #=>all the values are set to zero by default

Or you could use the implicit form and define your hash

grades = { "Jane Doe" => 10, "Jim Doe" => 6 }

In most cases, you are trying to create a structure that will be repeated. For example an array of hashes. each of these hashes are going to have the same form. Therefore, it is best practice to use a symbole instead of a literal as the key. A symbole is defined ONLY ONCE in the execution of your program. As a result, this will be taking significantly less amount of space on the disk, especially if your array grows and have a large amount of objects in it.

options = { :font_size => 10, :font_family => "Arial" }

But since this form was really common to use, Ruby has a syntactic sugar for it. This is very similar to the way that you define objects in JavaScripts.

options = { font_size: 10, font_family: "Arial" }

Hashes are commonly used for defining objects. They are passed into the method as an argument, and if they are the last argument passed to the method, no braces are needed

Person.create(name: "John Doe", age: 27)
