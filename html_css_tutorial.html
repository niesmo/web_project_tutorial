Introduction:

HTML is interpreted by browsers to create webpages. HTML is a very forgiving langauge, however authors should be strict when using it. CSS is a declarative language used to style HTML elements to look pretty!! You can easily get started by just using a text editor such as Notepad and any browser you wish!

HTML

  • Introduction: HTML is a universally used language for creating web components that are interpreted by many different broswers.
  • Basic elements used to start using HTML
    • html - outer tag; implies HTML document
    • head - inside <head> tag; header information goes here
    • body - inside <head> tag, below <head> tag, content goes here
    • p - inside <body> tag; basic content paragraph goes here
  •                         
Basic Elements of an HTML Page: Renders a page with just "This is my paragraph."
<html>
	<head>
		<title>HTML Basic Example</title>
		<meta charset="utf-8"/>
	</head>
	<body>
		<p>This is my paragraph.</p>
	</body>
</html>
    Try it
  • Some other common elements: in <body> tag
    • br - provides a break between lines
    • a - anchor tag to a hyperlink
    • hr - puts a break with horizontal line
    • img - allows to link to an image tag
    • div - defines a section in an HTML page
    • ul/li - unordered list with elements defined by li
    • table/tr/td - table with rows by tr and data by td
    • form - creates a place for user input (textboxes, checkboxes, etc)
  • Some other common elements: in <head> tag
    • title - tag to declare website name
    • meta charset="utf--8" - allows you to insert unicode
  • Unicode - allows certain elements to be expressed using numbers
  • Comments - a comment in HTML looks like :

CSS

  • Introduction - CSS is a way that HTML can be styled by specifying different elements and setting certain attributes to those elements.
  • Style Tag - The style tag is a way to declare CSS stylings in a HTML document either in-line or at the top of the page in the header section.
  • Linking to CSS Stylesheet - The other place CSS stylings can be kept is in a separate CSS file that is linked to the main page like such:
                
<html>
	<head>
		<link type="text/css" rel="stylesheet" href="3901Style.css" media="screen"/>
	</head>
	<body>
		<p> I want this styled a certain way </p>
	</body>
</html>
  • How to Style in a Separate Sheet
                
All em elements wil be red with a gray background
em {
	color:red;
	background: gray;
}

  • Box Model - Automatic box surrounding any html element that consists of padding, a border,and a margin.
    • Margin - Outside layer of space between elements
    • Padding - Inside layer of space between elements
    • Vertical Collapsing - Margins collapse to the size of the largest one. If a paragraph elements had a bottom margin of 25px and another paragraph element had a top margin of 50px, they would collapse so that there is only 50px between the two paragraph elements.
  • Inheritance - some stylings are passed down from parent tag to child tag. (font weight, color, family etc.)
  • Importance - some elements are selected by multiple tags when styling. There is a ranking system to help determine which styling should apply.
    • Location - which file is the stying found in. (Author, user, browser)
    • Marked As Importance - is the styling marked as !Important
    • Specificity - how specific is the styling
    • Declaration Order - what order is the styling defined in
  • Source of Stylings
    • Author - linked stylings by author
    • User - user settings interpreted by the browser
    • Browser - general stylings for default when not specified
  • Priority Order
    1. User Important - Most important!li>
    2. Author Important
    3. Author (normal)
    4. User (normal)
    5. Browser (normal) - Least important!
  • Unique Elements: ID's A specific name can be chosen if an ID is used. This name has to be specific to ONE specific element, meaning that it can't be used more than once. Specified inline with the tag id="one"
  • Block Elements - elements that have padding and margins around them that affect the layout of the document.
  • Special Tags - specifically the span tag. Used to style words within a paragraph that are not of a different element type.
  • Selecting Ancestors/Siblings - elements can be chosen as ancestors or siblings
    • Space between elements vs. comma
    • parent > child - select a child
    • element + first sibling
    • element ~ all siblings after
  • ICE order for specificity (Number of: ID's, Classes, Elements)
    • div.warning p {color: red} Sp = (0, 1, 2)
    • em {color: red} Sp = (0, 0, 1)
    • #caution p {color: red} Sp = (1, 0, 1)
    • div#caution p {color: red} Sp = (1, 0, 2)
    • The most specific is the last one!! Highest number of ID's
  • Pseudo Classes - Allows elements to be selected by attributes that are not elements
    • visited - :visited
    • active - :active
    • hover - :hover
    • first/last child - :first-child, :last-child
    • checked - :checked
  •
